# Azure 混合身分整合實作記錄

## 実施日 
2026-03-19

---

## 目標 

地端 Active Directory と Azure Entra ID を連携させ、
混合クラウド環境における身分管理の基礎を構築する。

---

## 構成 
```
地端（On-Premises）
└── Windows Server + Active Directory DC
    └── Entra Connect（同期ツール）
        ↓
Azure
└── Entra ID（クラウド身分管理）
└── VPN Gateway（ネットワーク接続）
```

---

## 実際に解決した問題

### 問題1：DC昇格失敗

- 現象：新しいフォレスト作成時にエラーが発生
- 原因：DNSが自分自身を指していなかった
- 解決：ネットワークアダプターのDNSを127.0.0.1に設定してから昇格
- 学び：AD構築はDNS設定が前提条件になる

---

### 問題2：Entra Connectインストール失敗

- 現象：インストール時に「先前未完成の案例」エラー
- 原因：SQL Server LocalDBの残留コンポーネントが競合
- 解決：コントロールパネルから残留コンポーネントを手動削除後に再インストール
- 学び：インストール失敗時は環境のクリーンアップが必要

---

### 問題3：同期失敗（Access Denied）

- 現象：Administratorでも同期時にアクセス拒否
- 原因：ADオブジェクトの権限継承が無効になっていた
- 解決：ADユーザーとコンピューターで権限継承を再有効化
- 学び：AD権限モデルがEntra Connect同期に影響する

---

### 問題4：ファイル転送（ネットワーク制限）

- 現象：学校ネットワークでツールのダウンロードが制限
- 解決：SMB管理者共有（\\IP\c$）で別PCからファイルを転送
- 学び：環境制限時の代替手段としてSMB共有が有効

---

### 問題5：VPN Gateway設定

- 現象：パラメータの意味が不明確
- 実施内容：
  - Public IP（1.160.12.23）= 教室全体の出口IP
  - Address Space（192.168.25.0/24）= 地端VMのネットワーク
  - 証明書をPowerShellでBase64変換してAzureに貼り付け
- 現状：Azure側でVPN Gatewayが正常表示。ルーティングの詳細は学習中
